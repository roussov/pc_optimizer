# Guide de contribution au projet PC Optimizer Console

Merci de votre intérêt pour contribuer à ce projet open-source !

Pour que vos contributions soient intégrées efficacement, merci de suivre les bonnes pratiques ci-dessous.

---

## Comment contribuer ?

### 1. Forker le projet

- Créez un fork du dépôt sur GitHub
- Clonez votre fork localement

### 2. Créer une branche

- Travaillez toujours dans une branche dédiée (ex : `feature/ajout-scan` ou `fix/menu-bug`)

### 3. Développer

- Suivez le style de code existant
- Ajoutez des commentaires clairs
- Testez vos modifications localement

### 4. Commit

- Faites des commits atomiques et bien décrits
- Utilisez un message clair et concis, par exemple :
  `fix: correction du bug dans le menu principal`
  `feat: ajout du module defrag`

### 5. Pull Request

- Ouvrez une pull request (PR) depuis votre branche vers la branche principale du projet  
- Expliquez clairement les changements apportés et pourquoi ils sont utiles  
- Soyez prêt·e à discuter et ajuster votre PR si nécessaire  

---

## Règles générales

- Respectez la licence GPLv3  
- Ne soumettez pas de code non testé ou incomplet  
- Soyez courtois·e et constructif·ve dans les échanges  
- N’hésitez pas à poser des questions dans les issues  

---

## Tests et Qualité

- Compilez et testez votre code sous Windows (MinGW, MSYS2)  
- Vérifiez que le menu fonctionne et que les modules s’exécutent sans erreurs  

---

Merci encore pour votre contribution, elle fait vivre ce projet !
